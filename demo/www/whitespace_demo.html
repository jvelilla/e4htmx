Glimmer Templates: Whitespace Control

Clean up unwanted blank lines and indentation in your output HTML or text templates. By placing hyphens adjacent to template delimiters ({{- and -}}), you can trim the trailing or leading whitespace of adjacent static text blocks.

⬅️ Left Trimming ({{-)

Adding a hyphen immediately after the opening brace removes all trailing whitespace (spaces, tabs, newlines) from the static text node immediately preceding the tag.

➡️ Right Trimming (-}})

Adding a hyphen immediately before the closing brace removes all leading whitespace (spaces, tabs, newlines) from the static text node immediately following the tag.

🧹 Clean Indentation

Allows you to write readable, nicely-indented nested loops and conditionals in your source templates without injecting extra carriage returns or double spacing into the output.

Whitespace Syntax Reference

Left Trim Delimiter

{{-

Tells the parser to search the static text directly on the left of this block tag and trim all its trailing whitespace.

Right Trim Delimiter

-}}

Tells the parser to search the static text directly on the right of this block tag and trim all its leading whitespace.

Block-only Rules

delimiters

Whitespace control is supported on double-brace block tags (like if, each, end, include, fill, etc.), avoiding expression conflicts in variables.
