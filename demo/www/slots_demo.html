Glimmer Templates: Slot Support

Enable true component composition and content projection. Declare placeholder areas using slots and fill them dynamically from the calling template, preserving variable isolation.

🎯 Slot Declarations

Use the {{slot name}} tag inside a component to declare a placeholder area where custom HTML content can be projected by callers.

📥 Content Projection

Define calling blocks using the nested {{include component}} syntax. Inside, use {{fill name}}...{{end}} to project HTML blocks.

🔒 Scoped Context Safety

Fill blocks resolve variables in the parent context where they are defined, whereas components render in their own scope. Slot boundaries prevent context bleed.

Slots Syntax Reference

Slot Declaration

slot

Declares a named target area inside a component template where callers can insert custom markup.

Syntax: {{slot header}}
Syntax: {{slot content}}
Declared inside partials. Unfilled slots evaluate to an empty string.

Content Fill

fill

Provides the custom HTML markup to project into the matching slot by name.

Syntax: {{fill header}}...{{end}}
Declared nested inside a block include. Must be closed with {{end}}.

Block Include

include

Includes a partial and wraps fill projection blocks. Requires a matching closing tag.

Syntax: {{include comp}}...{{end}}
Parsed as a block include if immediately followed by {{fill}} block tags.
