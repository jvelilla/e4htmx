Glimmer Templates: Pragmatic DbC

Eiffel's Design by Contract (DbC) principles map beautifully to template rendering. Enforce variable preconditions before rendering begins and inspect active scopes dynamically during development.

The Core Intent: Defensive vs. Assertive

🛡️ Input Validation (Defensive)
External Trust (Boundary checks)

Assumes the incoming data is untrusted, hostile, or messy. It is designed to filter out bad user input, malicious payloads, or simple typos (e.g., checking if an email has an @ symbol). It expects failures as a normal part of daily operations.

  • Normal Flow: User errors are routine; the application stays running.
  • Response: Return soft, friendly error messages to let the user correct their input.
  • Eiffel Principle: Intercept errors at the boundary. The backend routine is never called with invalid data.
💥 Design by Contract (Assertive)
Internal Trust (API preconditions)

Assumes the internal system components are bug-free. A precondition defines the exact state the software architecture must be in for a component to function. If a precondition fails, it means there is a programming bug in your code, not that the user typed something wrong.

  • Programming Bug: Fails fast; client developer failed to ensure preconditions.
  • Response: Raise an exception/violation alert immediately to the developer team.
  • Eiffel Principle: Enforces contract correctness. Preconditions are assertions, not input validation.

Core DbC Syntax Reference

Variable Presence

require

Asserts that required variables are defined and non-void in the render context.

Syntax: {{require username}}
Syntax: {{require user, role, items}}
Fails if any listed variable is missing or evaluates to Void.

Expression Assertions

require

Evaluates custom boolean preconditions before rendering content blocks.

Syntax: {{require age >= 18}}
Syntax: {{require exists token}}
Reuses Glimmer's logical expression parser to validate complex states.

State Inspection

dump

Inspects any context variable or the entire context scope using reflection.

Syntax: {dump user}
Syntax: {dump_context}
Uses Eiffel reflection (INTERNAL) to render interactive tables.
