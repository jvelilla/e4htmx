Glimmer Templates: Template Inheritance

Organize layout structures cleanly using multi-level inheritance. Declare blocks in layouts, override them in page templates, and inherit default content when no override is present.

🏗️ Extends Layout

Declare which layout template to inherit using the {{extends layout_name}} tag at the very top of your child template.

📦 Named Blocks

Define overrideable regions using {{block name}}...{{end}}. Children can replace the contents of these blocks by declaring blocks with the same name.

🔗 Multi-level Chains

Inheritance can span multiple levels (e.g. child extends layout, which extends base). Default block content in parent layouts is automatically used if not overridden.

Inheritance Syntax Reference

Extends Tag

extends

Declares which parent template layout structure to inherit from. Must be at the top of the file.

Syntax: {{extends base_layout}}
Parent layouts can be partials or relative template files on disk.

Block Declaration

block

Defines a named block with default content in the parent layout.

Syntax: {{block content}}...{{end}}
If a child does not override it, the default content inside the block will render.

Block Override

override

Declared in child templates to replace the block content in layout pages.

Syntax: {{block content}}Child HTML{{end}}
Replaces the matching parent block's body when rendering the hierarchy.
