Glimmer Templates: Component Composition

Assemble complex, responsive interfaces safely using Glimmer's Parameterized Include system. Leverage scoped child contexts to ensure complete variable isolation and prevent template side effects.

📦 Parameterized Includes

Pass explicit key-value properties into partials using the with syntax. Components receive resolved parent variables or literal parameters mapping directly to their child scope.

🔒 Scoped Context Isolation

Partials render in an isolated render context. They cannot access parent variables unless they are explicitly passed, preventing variable leakage and making component boundaries clear and modular.

📜 Component Contracts

Place {{require var}} assertions at the top of your components. If a required property is not passed, Glimmer fails fast in development, surfacing the bug immediately.

Component Syntax Reference

Variables & Constants

Include

Pass string constants (quoted) or parent variables directly into parameters.

Syntax: name=user_name
Syntax: role="Lead Developer"
Supports passing variables, dotted paths, numbers, booleans, and strings.

Scope Isolation

Scoping

Variables in the parent scope are not visible inside components unless explicitly mapped.

Parent Var: company
Inside Component: {company} → renders as raw text (empty)
Keeps components predictable and reusable across different contexts.

Boundary assertions

Contracts

Declare required inputs using preconditions at the very top of component files.

Precondition: {{require name}}
Fails Fast: returns 422 on breach.
Ensures type and presence agreements between templates and handlers.
